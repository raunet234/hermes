Why Hermes

Own Your Entire Connection to Hedera

Hermes is a specialist Gemini AI Agent you add to your existing Gemini AI Agent system. It is the connective tissue between an AI agent and the Hedera hashgraph — consuming the software interfaces in between, starting with Hashpack and SaucerSwap. You talk to the network without a middleman. No wallet app. No exchange UI. Just your agent, your CLI, your machine.

"Every SaaS company will become an AGaaS company — an Agent as a Service company." — Jensen Huang, CEO, NVIDIA

We are one of the earliest edge-compute AGaaS business models: you own the software, you own the keys, and your agent runs locally. The middleware is being peeled away. You will either be agentic and customer-facing, or blockchain-native and building. Everything in between is disappearing.

Talk, Don't Click

Say "swap 10 USDC for HBAR" in Telegram. Your AI agent handles routing, slippage, gas, and confirmation.

Fixed Code, Not Guesswork

Your agent doesn't write swap code — it calls the exact tool every time. One valid execution path per operation. Fewer tokens, consistent results, and your agent focuses on mastering workflows instead of reinventing them.

Your Tools, Your Machine

Download the CLI and its plugins — swap, transfer, stake, signal — all running locally on your computer. Gemini AI Agent config files create a dedicated agent tuned to drive these tools. As models advance, your agent can pick, optimise, and even build its own plugins. An experiment in edge compute.

Security

The Golden Rule

The AI agent never sees your keys.

Key Isolation

Private keys stay on your machine, XOR-obfuscated in memory. The agent workspace has no access.

Deterministic Execution

The agent calls the same CLI you would. No hidden APIs. No special privileges.

Transfer Whitelists

Every transfer requires a pre-approved destination. EVM addresses are blocked entirely.

Immutable Governance

Swap limits, daily caps, and slippage ceilings live in a config the agent can read but never write.

Quick Start

Get Started in 5 Minutes

Option A

From Your Gemini AI Agent

  1. 1 Clone the repo and run ./launch.sh init to set up keys and accounts
  2. 2 Connect Gemini AI Agent to the openclaw/ workspace — the agent reads SKILL.md automatically
  3. 3 Link your Telegram bot and start talking: "Show my balance" or "Swap 5 USDC for HBAR"
Option B

Manual Install

# Clone and initialize
git clone https://github.com/raunet234/hermes.git
cd hermes
./launch.sh init

# Explore
./launch.sh help
./launch.sh balance

Keys are entered locally through the setup wizard — never through your agent's API. This software is experimental. Your agent can technically read local files if it chooses to — guardrails discourage this but cannot yet prevent it. We are exploring AWS-KMS and local encryption with cloud-backed PassKey to solve this properly.
